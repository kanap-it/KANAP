# Security Policy

## Reporting A Vulnerability

Please report security vulnerabilities privately.

Preferred channel:
- GitHub: use this repository's private vulnerability reporting (Security Advisories).

Fallback channel:
- Email: `support@kanap.net`
- Subject: `[SECURITY] <short summary>`

Please do **not** open public GitHub issues for security vulnerabilities.

## What To Include

Include as much of the following as possible:
- A clear description of the issue and affected component(s)
- Reproduction steps or a proof of concept
- Impact assessment (what an attacker can do)
- Affected version/commit and environment details
- Suggested mitigation (if available)

## Response Targets

We aim to:
- Acknowledge receipt within **48 hours**
- Provide an initial triage update within **7 days**
- Share remediation status and expected timeline as work progresses

Complex issues may require more time, but we will keep communication active.

## Disclosure Policy

- We follow coordinated disclosure.
- After a fix is available, we may publish a security advisory with impact, affected versions, and remediation guidance.
- Reporters will be credited unless they prefer to remain anonymous.

## Safe Harbor

If you act in good faith, avoid privacy violations/service disruption/data destruction, and report findings promptly, we will not pursue legal action for your research.
